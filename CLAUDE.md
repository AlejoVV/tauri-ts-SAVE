## Project Overview

**Herramienta SAVE** — A Tauri desktop app for agricultural laboratory test management. Built with React + TypeScript frontend and Supabase as backend. The UI shows work orders, efficacy tests, lab montages, and generates DOCX reports.

## Commands

```bash
npm run dev        # Vite dev server (port 1420)
npm run tauri dev  # Run full Tauri desktop app (requires npm run dev in parallel)
npm run build      # tsc + vite build
npm run tauri build # Package as installable desktop app
```

No test runner is configured.

## Architecture

### Module System (`src/modulos/`)

Feature code lives in `src/modulos/<nombre-modulo>/` with a consistent internal layout:
```
modulo/
├── componentes/   # React components (barrel export via index.ts)
├── hooks/         # Custom hooks (useXxx.ts)
├── servicios/     # Supabase queries and mutations (index.ts)
├── tipos/         # TypeScript interfaces (index.ts)
└── utils/         # Pure utility functions
```

Active modules: `nucleo`, `eficacia`, `informes`, `ordenes-trabajo`, `resumen`, `objetivos`.
Stub modules (not yet implemented): `analiticas`, `contactos`, `fincas`, `laboratorio`, `productos`, `gestionar`.

### Navigation / Routing

There is **no React Router**. `App.tsx` holds an `activeView` string state and renders one module at a time via a switch. `BarraLateral.tsx` (`nucleo/componentes/layout/`) drives navigation by calling `onViewChange`.

Current views: `objetivos`, `maestra`, `eficacia`, `informes`, `resumen`.

### State Management

- **Server state:** React Query (`@tanstack/react-query`). Queries use `useQuery`, mutations use `useMutation` with `queryClient.invalidateQueries` on success.
- **Local UI state:** `useState` inside components (modals, form fields, snackbars).
- No global client-state store (Redux/Zustand/etc.).

### Supabase

- **Client:** `src/modulos/nucleo/lib/supabaseClient.ts` — exports `supabase` (main), `supabaseTest`, `supabaseProduction`. Environment toggles via `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`.
- **Types:** `src/modulos/nucleo/lib/supabase.ts` — auto-generated TypeScript types for all tables/views (do not edit by hand; regenerate with `supabase gen types`).
- **Key views used across modules:** `vistamaestratotal`, `vistamaestra`, `vistaobjetivosprecios`, `vistacontactoscompanias`.
- **Pagination pattern:** Services fetch in loops of 1000 rows using `.range(from, from + pageSize - 1)`.

### Data Tables

`MaterialReactTable` (MRT v3) is used for all main data tables. Column definitions are memoized with `useMemo`. Tables support row selection, filtering, sorting, and inline editing tied to mutations.

### Document Generation

Reports are generated by calling the Supabase Edge Function `fill-docx-template` (defined in `supabase/functions/fill-docx-template/`). The function receives test data and returns a filled DOCX binary. The `docx` package and `file-saver` are also available for client-side generation.

### UI Components

- `src/components/ui/` — shadcn/ui primitives (button, card, dialog, input, etc.).
- Material UI (MUI v7) used alongside shadcn for complex components (table toolbars, date pickers, snackbars).
- Path alias `@/` maps to `src/`.
- `cn()` from `src/lib/utils.ts` is the standard class merging utility (`clsx` + `tailwind-merge`).

### Error Handling Convention

Services throw errors. Components catch them and surface feedback via MUI `Snackbar`/`Alert`:
```typescript
setSnackbar({ open: true, message: "...", severity: "error" })
```
